# 21. 合并两个有序链表

## 题目信息

- 难度：简单
- 链接：https://leetcode.cn/problems/merge-two-sorted-lists/
- 主知识点：链表
- 相关知识点：双指针、虚拟头节点、递归
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 两个链表都已经按非递减顺序排列
- 要把两个链表合并成一个新的有序链表
- 不需要创建新值节点，可以直接复用原链表节点
- 链表头节点可能变化，适合用虚拟头节点统一处理

## 解题思路

### 暴力思路

可以把两个链表的值全部取出来放进数组，排序后再重新建链表。但这样没有利用两个链表本身已经有序的条件，而且额外空间更大。

### 优化关键

两个链表都已经有序，所以每次只需要比较 `list1` 和 `list2` 当前头节点的值，把较小的节点接到结果链表后面。这个过程和归并排序里的 merge 步骤一样。

链表操作里最容易麻烦的是“结果链表的第一个节点怎么接”。用一个虚拟头节点 `prehead` 可以避免单独判断头节点是否为空，真正的答案从 `prehead->next` 开始。

### 最终做法

1. 创建虚拟头节点 `prehead`
2. 用 `prev` 指向结果链表当前尾节点
3. 当两个链表都不为空时，比较当前节点值
4. 把较小节点接到 `prev->next`
5. 移动被接走的链表指针，并移动 `prev`
6. 循环结束后，把非空的剩余链表直接接到 `prev->next`
7. 返回 `prehead->next`

## C++ 代码讲解

```cpp
class Solution {
public:
    ListNode* mergeTwoLists(ListNode* list1, ListNode* list2) {
        ListNode* prehead = new ListNode(-1);
        ListNode* prev = prehead;

        while (list1 != nullptr && list2 != nullptr) {
            if (list1->val < list2->val) {
                prev->next = list1;
                list1 = list1->next;
            } else {
                prev->next = list2;
                list2 = list2->next;
            }
            prev = prev->next;
        }

        prev->next = list1 == nullptr ? list2 : list1;
        return prehead->next;
    }
};
```

关键点：

- `prehead`：虚拟头节点，不属于最终答案，只是为了统一接链表的逻辑
- `prev`：结果链表的尾指针，永远指向已经合并部分的最后一个节点
- `list1`、`list2`：分别指向两个输入链表当前还没合并的第一个节点
- `prev->next = list1/list2`：不是复制节点，而是把原链表节点接到结果链表后面
- `prev = prev->next`：每接上一个节点，尾指针必须跟着前进
- `prev->next = list1 == nullptr ? list2 : list1`：剩下的链表本来就是有序的，可以整段接上

循环不变量 / 状态含义：

- `prehead->next` 到 `prev` 是已经合并好的有序链表
- `prev` 始终指向合并链表的最后一个节点
- `list1` 和 `list2` 指向各自尚未合并部分的头节点
- 每次循环都会消耗 `list1` 或 `list2` 的一个节点

边界处理：

- 如果 `list1` 一开始为空，循环不进入，直接返回 `list2`
- 如果 `list2` 一开始为空，循环不进入，直接返回 `list1`
- 如果两个都为空，`prehead->next` 是 `nullptr`，返回空链表
- 你这里用 `new ListNode(-1)` 创建虚拟头节点，LeetCode 可以通过；实际工程里更常用栈对象 `ListNode dummy(-1)`，避免手动释放问题

## 复杂度分析

- 时间复杂度：O(m + n)，两个链表的每个节点最多访问一次
- 空间复杂度：O(1)，只使用常数个指针，不计输入链表本身

## 面试讲法

可以这样讲：

这题和归并排序里的合并步骤一样。因为两个链表本来就是有序的，我维护一个结果链表的尾指针，每次比较两个链表当前节点，把较小的节点接到结果链表后面。为了避免单独处理结果链表头节点，我使用一个虚拟头节点，最后返回 `dummy.next` 或 `prehead->next`。当其中一个链表走完后，另一个链表剩下的部分已经有序，可以直接接到结果链表末尾。

可能追问：

- 为什么可以直接接剩余链表：剩余部分内部仍然有序，而且所有已经接入的节点不大于当前比较后留下的节点
- 是否需要创建新节点：不需要，直接复用原链表节点即可
- 如果两个值相等怎么处理：接任意一个都能保持非递减顺序；代码里相等时接 `list2`
- 能不能递归：可以，递归写法更短，但递归栈空间是 O(m + n)

## 模板沉淀

- 是否沉淀模板：是
- 对应模板：[虚拟头节点合并链表](../templates/cpp/linked-list.md)
- 可复用点：当链表结果的头节点不确定时，用虚拟头节点 + 尾指针统一尾插

## 易错点

- 忘记 `prev = prev->next`，会导致一直覆盖同一个 `next`
- 移动 `list1/list2` 前要先把当前节点接到 `prev->next`
- 循环条件是两个链表都不为空：`list1 != nullptr && list2 != nullptr`
- 循环结束后不要继续一个个比较，直接接上剩余链表
- 返回的是 `prehead->next`，不是 `prehead`
- 实际工程中如果用 `new` 创建虚拟头节点，需要考虑释放；刷题里可改成栈对象避免这个问题

## 复盘卡片

- 一句话记忆：合并有序链表 = 虚拟头节点开路，`prev` 永远指向结果链表尾部
- 下次先想：链表头会不会变化；如果会，优先考虑 dummy/prehead
- 如果写错，优先检查：`prev` 是否前进、剩余链表是否接上、返回值是否是 `prehead->next`
