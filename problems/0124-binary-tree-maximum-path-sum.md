# 124. 二叉树中的最大路径和

## 题目信息

- 难度：困难
- 链接：https://leetcode.cn/problems/binary-tree-maximum-path-sum/
- 主知识点：二叉树
- 相关知识点：DFS、后序遍历、树形 DP
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 路径可以从任意节点开始，到任意节点结束
- 路径不一定经过根节点
- 每个节点在一条路径中最多出现一次
- 对父节点来说，子树只能向上贡献一条“单边路径”
- 但更新全局答案时，可以同时使用左子树贡献和右子树贡献

## 解题思路

### 暴力思路

可以枚举每个节点作为路径的最高拐点，然后分别计算左右子树能提供的最大路径贡献。但如果每次都重新计算子树贡献，会重复遍历很多节点，时间复杂度可能退化到 O(n^2)。

### 优化关键

用后序 DFS，让每个节点只计算一次。

递归函数 `dfs(node)` 的含义是：

```text
从 node 出发，向下走到某个后代节点，能提供给父节点的最大单边路径和。
```

为什么只能返回单边？

因为父节点如果要继续往上连接路径，从当前节点这里最多只能选择左边或右边一条链。否则如果同时带上左右两边，再连接父节点，就会分叉，不再是一条合法路径。

但是更新全局答案时，当前节点可以作为路径的最高拐点，此时可以同时使用：

```cpp
leftVal + node->val + rightVal
```

如果某个子树贡献是负数，就不要带上它，所以返回给父节点时和 `0` 取最大值。

### 最终做法

1. 用全局变量 `ans` 记录最大路径和，初始化为 `INT_MIN`
2. 写后序 DFS
3. 空节点向上贡献 `0`
4. 递归计算左右子树的最大单边贡献
5. 用 `leftVal + rightVal + node->val` 更新全局答案
6. 返回 `max(max(leftVal, rightVal) + node->val, 0)` 给父节点
7. 调用 `dfs(root)` 后返回 `ans`

## C++ 代码讲解

```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
public:
    int ans = INT_MIN;

    int dfs (TreeNode* node) {
        if (node == nullptr) {
            return 0;
        }

        // 左右子树能向当前节点提供的最大单边贡献
        int leftVal = dfs(node->left);
        int rightVal = dfs(node->right);

        // 当前节点作为路径最高拐点时，可以同时连接左边、当前节点、右边
        ans = max(ans, leftVal + rightVal + node->val);

        // 返回给父节点时只能选择一边；如果贡献为负，就直接丢弃，返回 0
        return max(max(leftVal, rightVal) + node->val, 0);
    }

    int maxPathSum(TreeNode* root) {
        dfs(root);
        return ans;
    }
};
```

关键点：

- `ans`：全局最大路径和，路径可以出现在任意位置
- `dfs(node)`：当前节点向父节点提供的最大单边贡献
- `leftVal/rightVal`：左右子树向当前节点提供的最大单边贡献
- `leftVal + rightVal + node->val`：以当前节点作为拐点的完整路径和
- `max(max(leftVal, rightVal) + node->val, 0)`：返回给父节点的贡献，只能选一边，负数贡献丢弃

循环不变量 / 状态含义：

- 递归处理完一个节点后，以它为根的子树内部最大路径已经尝试更新到 `ans`
- `dfs` 返回值永远表示“可以继续向父节点连接”的单边路径贡献
- `ans` 可以记录左右双边路径，但 `dfs` 返回值不能是双边路径

边界处理：

- 题目保证至少有一个节点，所以 `maxPathSum(root)` 不需要处理空树
- `ans` 必须初始化为 `INT_MIN`，不能初始化为 `0`，因为节点值可能全是负数
- 空节点返回 `0`，表示不提供任何贡献
- 返回给父节点时如果结果为负，取 `0`，表示不选这条子路径

## 复杂度分析

- 时间复杂度：O(n)，每个节点只访问一次
- 空间复杂度：O(h)，其中 `h` 是树高，递归调用栈最坏为 O(n)

## 面试讲法

可以这样讲：

这题的关键是区分“全局答案”和“递归返回值”。路径可以在当前节点处从左子树经过当前节点再到右子树，所以更新答案时可以用 `left + node->val + right`。但是如果当前节点要把路径继续交给父节点，就不能同时带左右两边，只能选择左边或右边较大的一条单边路径。因此我的 DFS 返回当前节点能向父节点贡献的最大单边路径和。如果某个贡献是负数，就不要它，返回时和 `0` 取最大值。全局答案初始化为 `INT_MIN`，保证全负数树也能得到正确答案。

可能追问：

- 为什么返回值不能带左右两边：带左右两边再接父节点会形成分叉，不是一条合法路径
- 为什么负数贡献可以丢弃：路径至少包含当前节点，负贡献只会让路径和变小
- 为什么 `ans` 不能初始化为 0：如果所有节点都是负数，答案应该是最大的那个负数
- 这是不是树形 DP：是，`dfs(node)` 就是当前子树向父节点提供的最优状态

## 易错点

- `ans` 初始化不能是 `0`
- `dfs` 返回给父节点时不能同时返回左右两边
- 更新答案时要用 `leftVal + rightVal + node->val`
- 空节点返回 `0` 是因为它不贡献路径，不代表答案可以是空路径
- 如果全是负数，最终答案应该是最大节点值，而不是 `0`

## 复盘卡片
