# 20. 有效的括号

## 题目信息

- 难度：简单
- 链接：https://leetcode.cn/problems/valid-parentheses/
- 主知识点：栈
- 相关知识点：字符串、括号匹配
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 括号需要按正确顺序闭合
- 后出现的左括号，必须先被匹配
- 这种“后进先出”的匹配关系天然适合栈
- 字符串只包含括号，不需要处理其他字符

## 解题思路

### 暴力思路

可以反复删除字符串中的 `"()"`、`"[]"`、`"{}"`，直到不能删除为止，最后看字符串是否为空。但这种做法实现麻烦，反复扫描也不够直接。

### 优化关键

括号匹配的本质是：最近遇到的左括号，必须最先被对应的右括号关闭。

用栈维护“还没匹配的括号”。你的写法更进一步：遇到左括号时，不压入左括号本身，而是压入它期待匹配的右括号：

- 遇到 `'('`，压入 `')'`
- 遇到 `'['`，压入 `']'`
- 遇到 `'{'`，压入 `'}'`

遇到右括号时，只要检查它是否等于栈顶期待的字符即可。

### 最终做法

1. 创建栈 `st`
2. 从左到右遍历字符串
3. 遇到左括号，把对应的右括号压入栈
4. 遇到右括号时：
   - 如果栈为空，说明没有左括号能匹配，返回 `false`
   - 如果栈顶不等于当前右括号，说明类型或顺序错误，返回 `false`
   - 否则弹出栈顶，完成一次匹配
5. 遍历结束后，只有栈为空才说明所有左括号都被匹配

## C++ 代码讲解

```cpp
class Solution {
public:
    bool isValid(string s) {
        stack<char> st;

        for (char c : s) {
            if (c == '(') {
                st.push(')');
            } else if (c == '[') {
                st.push(']');
            } else if (c == '{') {
                st.push('}');
            } else {
                if (st.empty() || st.top() != c) {
                    return false;
                }
                st.pop();
            }
        }

        return st.empty();
    }
};
```

关键点：

- `stack<char> st`：保存当前还期待出现的右括号
- 左括号分支：把匹配它的右括号压栈
- `st.empty()`：当前是右括号，但前面没有可匹配的左括号
- `st.top() != c`：当前右括号不是最近左括号期待的类型
- `st.pop()`：当前右括号匹配成功，弹掉这个期待
- `return st.empty()`：遍历结束后，如果栈里还有期待的右括号，说明有左括号没闭合

循环不变量 / 状态含义：

- 遍历到任意位置时，`st` 中保存的是前缀中所有未匹配左括号对应的期待右括号
- 栈顶是当前最先应该出现的右括号
- 一旦当前右括号和栈顶不匹配，括号顺序或类型就不合法

边界处理：

- 字符串以右括号开头时，栈为空，直接返回 `false`
- `"([)]"` 会在读到 `')'` 时发现栈顶期待的是 `']'`，返回 `false`
- `"((("` 遍历中不会失败，但最后栈不为空，返回 `false`
- 题目保证字符串只包含括号，所以 `else` 分支可以认为一定是右括号

## 复杂度分析

- 时间复杂度：O(n)，每个字符最多入栈或出栈一次
- 空间复杂度：O(n)，最坏情况下所有字符都是左括号

## 面试讲法

可以这样讲：

括号匹配要求最近打开的左括号最先闭合，所以它是典型的后进先出问题，用栈处理。遍历字符串时，如果遇到左括号，我直接把它期待的右括号压入栈；如果遇到右括号，就检查栈是否为空以及栈顶是否等于当前字符。不匹配就返回 false，匹配就弹栈。最后如果栈为空，说明所有括号都正确闭合。

可能追问：

- 为什么用栈：因为括号嵌套要求最近的左括号先被关闭
- 为什么压右括号而不是左括号：压期待的右括号后，遇到右括号可以直接和栈顶比较
- 最后为什么还要检查栈空：可能存在未闭合的左括号
- 如果字符串包含其他字符怎么办：按题意这里没有；如果有，需要决定忽略还是判非法

## 易错点

- 遇到右括号时，必须先判断栈是否为空，再访问 `st.top()`
- 最后不能直接返回 `true`，要检查 `st.empty()`
- `"([)]"` 这种顺序错误不能只看数量，要靠栈顶匹配
- 压期待的右括号时，映射关系不能写反
- 题目里的括号是字符，要用单引号，例如 `'('`

## 复盘卡片

- 一句话记忆：有效括号 = 左括号压入期待的右括号，右括号必须等于栈顶
- 下次先想：嵌套匹配就是后进先出
- 如果写错，优先检查：空栈判断、栈顶匹配、最后是否栈空
