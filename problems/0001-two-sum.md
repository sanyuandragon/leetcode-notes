# 1. 两数之和

## 题目信息

- 难度：简单
- 链接：https://leetcode.cn/problems/two-sum/
- 主知识点：哈希表
- 相关知识点：数组、查找、空间换时间
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 要在数组里找两个数，使它们的和等于 `target`
- 需要返回的是下标，不是数值本身
- 每个输入只对应一个答案，且不能重复使用同一个元素
- 暴力解法里存在“对当前数，查找另一个数”的动作

## 解题思路

### 暴力思路

最直接的做法是两层循环枚举所有数对 `(i, j)`，判断 `nums[i] + nums[j] == target`。这样时间复杂度是 O(n^2)，虽然能做，但没有利用“查找另一个数”这个步骤可以被优化的特点。

### 优化关键

当遍历到 `nums[i]` 时，如果要凑出 `target`，另一个数必须是：

```cpp
need = target - nums[i]
```

如果之前已经见过 `need`，就可以直接返回之前记录的下标和当前下标。为了快速判断一个值是否出现过，用哈希表把“值 -> 下标”存起来。

关键顺序是：**先查，再插入当前数**。这样可以避免当前元素和自己配对。

### 最终做法

1. 创建哈希表 `mp`，记录已经遍历过的数值及其下标
2. 从左到右遍历数组
3. 对当前数 `nums[i]`，计算需要的补数 `need = target - nums[i]`
4. 如果 `need` 已经在哈希表中，返回 `{mp[need], i}`
5. 否则把当前数和下标插入哈希表
6. 理论上题目保证有解，最后的 `return {}` 只是兜底

## C++ 代码讲解

```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
       unordered_map<int, int> mp;
       int n = nums.size();

       for (int i = 0; i < n; ++i) {
            int need = target - nums[i];
            if (mp.find(need) != mp.end()) {
                return {mp[need], i};
            }
            mp.insert({nums[i], i});
       }

       return {};
    }
};
```

关键点：

- `unordered_map<int, int> mp`：键是数组值，值是该数组值出现过的下标
- `need = target - nums[i]`：把“两数之和”转成“查找补数”
- `mp.find(need) != mp.end()`：判断补数是否已经出现过
- `return {mp[need], i}`：补数下标在前，当前下标在后
- `mp.insert({nums[i], i})`：把当前数存入哈希表，供后面的元素匹配

循环不变量 / 状态含义：

- 遍历到下标 `i` 时，`mp` 中只保存了 `0...i-1` 的元素
- 因为当前元素还没插入，所以不会出现自己和自己配对
- 一旦找到 `need`，就说明 `nums[mp[need]] + nums[i] == target`

边界处理：

- 示例 `[3, 3]`、`target = 6` 可以正确返回 `{0, 1}`，因为第一个 `3` 先插入，第二个 `3` 再找到它
- `mp.insert({nums[i], i})` 遇到重复 key 时不会覆盖旧下标；本题只要求任意一个答案，这样是可以的
- 如果想记录某个值最近一次出现的位置，可以写成 `mp[nums[i]] = i`
- 题目保证存在唯一答案，所以 `return {}` 通常不会执行

## 复杂度分析

- 时间复杂度：O(n)，每个元素遍历一次，哈希查找均摊 O(1)
- 空间复杂度：O(n)，最坏情况下哈希表存下大部分元素

## 面试讲法

可以这样讲：

暴力做法需要枚举两个下标，时间是 O(n^2)。优化点在于，遍历到当前数时，另一个数其实已经确定为 `target - nums[i]`。所以我用哈希表记录之前出现过的数和下标。每次先查补数是否已经出现，如果出现就返回两个下标；如果没出现，再把当前数插入哈希表。先查后插可以保证不会重复使用同一个元素。

可能追问：

- 为什么不能先插再查：如果 `target = nums[i] * 2`，可能会把当前元素和自己配对
- 为什么哈希表存下标而不是只存值：题目要求返回下标
- 重复元素怎么办：先查后插能自然处理 `[3, 3]`
- 如果没有答案怎么办：本题保证有答案；不保证时返回空数组或约定错误值

## 模板沉淀

- 是否沉淀模板：是
- 对应模板：[哈希查找补数](../templates/cpp/hash-table.md)
- 可复用点：边遍历边维护“已经见过的值”，把 O(n) 查找降成 O(1)

## 易错点

- 不能返回数值，要返回下标
- 不要排序，排序会破坏原始下标；如果排序就需要额外保存原下标
- 要先查 `need`，再插入当前 `nums[i]`
- 判断存在性用 `find`，不要直接用 `mp[need]`，否则会插入默认值
- 使用 `insert` 时，重复 key 不会覆盖旧下标；本题没问题，但要知道这个行为

## 复盘卡片

- 如果写错，优先检查：是否先查后插、是否用 `find` 判断存在、是否返回下标
