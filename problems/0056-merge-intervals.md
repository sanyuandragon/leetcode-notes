# 56. 合并区间

## 题目信息

- 难度：中等
- 链接：https://leetcode.cn/problems/merge-intervals/
- 主知识点：区间
- 相关知识点：排序、贪心、数组
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 给出一组区间 `[start, end]`
- 要合并所有重叠区间
- 区间顺序不一定有序，需要先按左端点排序
- 扫描时只需要和答案中最后一个区间比较

## 解题思路

### 暴力思路

可以不断枚举两个区间，发现有重叠就合并，再重新扫描，直到没有区间可以继续合并。但这样每次合并后都可能要重新检查很多区间，代码复杂，效率也不稳定。

### 优化关键

先把所有区间按左端点升序排序。排序后，后面的区间左端点一定不会比前面的更小。

维护答案数组 `ans`：

- 如果当前区间和 `ans` 最后一个区间不重叠，就直接加入答案
- 如果当前区间和最后一个区间重叠，就更新最后一个区间的右端点

判断是否重叠：

```cpp
start <= ans.back()[1]
```

如果 `start > ans.back()[1]`，说明当前区间在最后一个区间右侧，且中间没有交集。

### 最终做法

1. 创建答案数组 `ans`
2. 对 `intervals` 排序，默认会先按左端点，再按右端点升序
3. 遍历每个区间
4. 取出当前区间的 `start` 和 `end`
5. 如果 `ans` 为空，或者当前区间左端点大于答案最后区间右端点，直接加入
6. 否则说明区间重叠，更新 `ans.back()[1]`
7. 返回 `ans`

## C++ 代码讲解

```cpp
class Solution {
public:
    vector<vector<int>> merge(vector<vector<int>>& intervals) {
        vector<vector<int>> ans;

        // 默认按每个 vector 的字典序排序：先按左端点，再按右端点
        sort(intervals.begin(), intervals.end());

        for (int i = 0; i < intervals.size(); ++i) {
            int start = intervals[i][0];
            int end = intervals[i][1];

            // 没有已有区间，或者当前区间和最后一个合并区间不重叠
            if (ans.empty() || start > ans.back()[1]) {
                ans.push_back({start, end});
            }else {
                // 有重叠，扩展最后一个区间的右端点
                ans.back()[1] = max(ans.back()[1], end);
            }
        }

        return ans;
    }
};
```

关键点：

- `sort(intervals.begin(), intervals.end())`：按左端点从小到大排序
- `ans`：已经合并好的不重叠区间
- `ans.back()`：当前可能与新区间重叠的最后一个区间
- `start > ans.back()[1]`：不重叠，需要开启新区间
- `ans.back()[1] = max(ans.back()[1], end)`：重叠时合并右端点

循环不变量 / 状态含义：

- 每次循环开始时，`ans` 中的区间已经按左端点排序且互不重叠
- 当前区间只可能和 `ans` 的最后一个区间发生重叠
- 处理完当前区间后，`ans` 仍然保持有序且互不重叠

边界处理：

- 题目保证 `intervals.length >= 1`
- 区间端点相接也算重叠，比如 `[1,4]` 和 `[4,5]` 应合并成 `[1,5]`
- 因此不重叠条件必须是 `start > ans.back()[1]`
- 如果写成 `start >= ans.back()[1]`，会错过端点相接的合并情况

## 复杂度分析

- 时间复杂度：O(n log n)，主要来自排序
- 空间复杂度：O(log n) 或 O(n)，取决于排序实现和返回结果；除答案外额外空间通常为 O(log n)

## 面试讲法

可以这样讲：

我先按区间左端点排序。排序后，从左到右扫描时，当前区间如果要发生重叠，只可能和已经合并结果里的最后一个区间重叠。因此我维护一个答案数组。当前区间左端点如果大于答案最后区间的右端点，就说明它们不重叠，直接加入答案；否则就合并区间，把最后一个区间的右端点更新为两者右端点的最大值。这样扫描一遍即可完成合并。

可能追问：

- 为什么先排序：不排序时无法保证只和最后一个合并区间比较
- 为什么只看 `ans.back()`：排序后，答案中更早的区间右端点不会越过最后一个区间
- 端点相等算不算重叠：本题算，所以 `[1,4]` 和 `[4,5]` 要合并
- 能不能原地合并：可以用原数组前部作为结果区间，但写法会更偏原地技巧

## 易错点

- 忘记排序
- 把端点相接误判为不重叠
- 不重叠条件是 `start > ans.back()[1]`
- 重叠时要更新右端点为最大值，而不是直接覆盖成 `end`
- 当前区间只需要和答案最后一个区间比较，不需要回头扫所有区间
