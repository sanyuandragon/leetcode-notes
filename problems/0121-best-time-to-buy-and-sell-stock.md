# 121. 买卖股票的最佳时机

## 题目信息

- 难度：简单
- 链接：https://leetcode.cn/problems/best-time-to-buy-and-sell-stock/
- 主知识点：贪心
- 相关知识点：数组、动态规划、前缀最小值
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 只能买卖一次
- 必须先买入，再在未来某一天卖出
- 要最大化 `卖出价格 - 买入价格`
- 遍历到某一天时，只需要知道它之前出现过的最低价格

## 解题思路

### 暴力思路

最直接的做法是枚举买入日 `i` 和卖出日 `j`，其中 `i < j`，计算 `prices[j] - prices[i]` 的最大值。这样是 O(n^2)，在 `prices.length` 最大为 `10^5` 时会超时。

### 优化关键

如果把第 `i` 天当作卖出日，那么最优买入日一定是 `0...i` 中价格最低的那一天。

所以从左到右遍历价格时，维护两个量：

- `minPrice`：到当前天为止见过的最低价格
- `maxProfit`：到当前天为止能获得的最大利润

每天都尝试“今天卖出”：

```cpp
profit = prices[i] - minPrice;
```

然后用它更新最大利润。最后再用当天价格更新历史最低价。

### 最终做法

1. 初始化 `minPrice = prices[0]`
2. 初始化 `maxProfit = 0`，表示不交易时利润为 `0`
3. 从左到右遍历每一天
4. 把当天当作卖出日，计算 `prices[i] - minPrice`
5. 用这个利润更新 `maxProfit`
6. 用当天价格更新 `minPrice`
7. 返回 `maxProfit`

## C++ 代码讲解

```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int minPrice = prices[0];
        int maxProfit = 0;

        for (int i = 0; i < prices.size(); ++i) {
            int profit = prices[i] - minPrice;
            maxProfit = max(maxProfit, profit);
            minPrice = min(minPrice, prices[i]);
        }

        return maxProfit;
    }
};
```

关键点：

- `minPrice`：遍历到当前天之前或当前天，见过的最低买入价格
- `maxProfit`：目前能获得的最大利润
- `profit = prices[i] - minPrice`：假设今天卖出，可以赚到的利润
- `maxProfit = max(maxProfit, profit)`：更新全局最优
- `minPrice = min(minPrice, prices[i])`：把今天也纳入之后卖出日可选择的买入价格

循环不变量 / 状态含义：

- 遍历完第 `i` 天后，`minPrice` 是 `prices[0...i]` 的最小值
- 遍历完第 `i` 天后，`maxProfit` 是在 `prices[0...i]` 内完成一次买卖能得到的最大利润
- 每一天都只需要尝试作为卖出日，不需要回头枚举所有买入日

边界处理：

- 题目保证 `prices.length >= 1`，所以 `prices[0]` 安全
- 如果价格一直下降，`profit` 最大也不会超过 `0`，最终返回 `0`
- 你的代码从 `i = 0` 开始遍历，此时 `profit = 0`，是合理的
- 如果想更强调“先买后卖”，也可以从 `i = 1` 开始遍历

## 复杂度分析

- 时间复杂度：O(n)，只遍历一次价格数组
- 空间复杂度：O(1)，只维护两个变量

## 面试讲法

可以这样讲：

这题只能交易一次，并且卖出日必须在买入日之后。遍历到某一天时，如果我选择今天卖出，那么最优买入价一定是今天之前见过的最低价格。因此我维护一个 `minPrice` 表示历史最低价，每天用 `prices[i] - minPrice` 尝试更新最大利润，然后再用今天的价格更新 `minPrice`。这样每一天只处理一次，时间复杂度是 O(n)。

可能追问：

- 为什么不用排序：排序会破坏时间顺序，股票必须先买后卖
- 为什么最大利润初始化为 `0`：如果没有任何赚钱机会，可以选择不交易
- 这是不是动态规划：可以理解成压缩后的 DP，`minPrice` 等价于维护买入后的最优状态
- 多次买卖怎么办：那是另一类股票 DP/贪心题，状态和约束会变化

## 易错点

- 不能排序，因为排序会改变买卖日期顺序
- `minPrice` 表示历史最低买入价，不是当前最低利润
- `maxProfit` 初始化为 `0`，因为可以不交易
- 只允许一次买卖，不要把多段上涨利润累加起来
- 如果从 `i = 0` 开始遍历，第一天利润为 `0`，不会影响答案

- 如果写错，优先检查：是否累加了多次利润、是否先买后卖、`maxProfit` 是否允许为 `0`
