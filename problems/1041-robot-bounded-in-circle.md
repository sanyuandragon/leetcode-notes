# 1041. 困于环中的机器人

## 题目信息

- 难度：中等
- 链接：https://leetcode.cn/problems/robot-bounded-in-circle/
- 主知识点：模拟
- 相关知识点：方向数组、数学、周期性
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 机器人在平面上移动，并且有固定的朝向
- 指令只有 `G`、`L`、`R`
- 题目问“无限重复执行指令后是否会被困在环中”
- 单轮指令执行后的位置和方向，决定了无限重复后的走势

## 解题思路

### 暴力思路

可以一直重复执行指令，记录机器人走过的位置，判断是否进入循环。但题目中的重复次数是无限的，不能真的模拟到无限次。

也可以模拟很多轮，比如 4 轮后看是否回到原点。这个方法能过，因为方向最多 4 种，但还可以进一步压缩成只模拟 1 轮。

### 优化关键

只执行一遍 `instructions` 后，有两种情况一定有界：

- 回到原点：已经完成一个闭环，后面不断重复同样的闭环
- 没回到原点，但方向不再朝北：每轮结束后的位移方向会旋转，最多 4 轮后整体位移抵消

只有一种情况无界：

- 没回到原点，并且方向仍然朝北

因为此时每一轮都会沿同一个方向产生相同位移，机器人会越走越远。

### 最终做法

1. 用 `dir` 表示当前方向：`0` 北、`1` 东、`2` 南、`3` 西
2. 用 `dx/dy` 数组表示四个方向上的位移
3. 从原点 `(0, 0)` 出发，执行一遍指令
4. 遇到 `G` 就按照当前方向移动
5. 遇到 `L` 就把方向左转 90 度
6. 遇到 `R` 就把方向右转 90 度
7. 最后判断：回到原点，或者方向不是北，就返回 `true`

## C++ 代码讲解

```cpp
class Solution {
public:
    bool isRobotBounded(string instructions) {
        int dir = 0;//0north 1east 2south 3west
        int dx[4] = {0, 1, 0, -1};
        int dy[4] = {1, 0, -1, 0};

        int x = 0, y = 0;

        for (char c : instructions) {
            if (c == 'G') {
                // 按当前方向前进一步
                x += dx[dir];
                y += dy[dir];
            }else if (c == 'L') {
                // 左转 90 度，相当于方向编号减 1；加 3 防止出现负数
                dir = (dir + 3) % 4;
            }else {
                // 右转 90 度，相当于方向编号加 1
                dir = (dir + 1) % 4;
            }
        }
        // 一轮后回到原点，或者朝向发生改变，都说明重复执行会被困在环中
        return (x == 0 && y == 0) || dir != 0;
    }
};
```

关键点：

- `dir`：当前朝向，`0/1/2/3` 分别表示北、东、南、西
- `dx/dy`：方向数组，和 `dir` 的编号一一对应
- `L`：左转等价于 `dir = (dir + 3) % 4`
- `R`：右转等价于 `dir = (dir + 1) % 4`
- `(x == 0 && y == 0)`：一轮后回到起点，说明形成闭环
- `dir != 0`：一轮后朝向改变，说明多轮后位移方向会旋转并抵消

循环不变量 / 状态含义：

- 执行到任意指令前，`(x, y)` 表示当前机器人位置
- `dir` 始终表示机器人当前面朝方向
- `dx[dir]` 和 `dy[dir]` 始终是当前方向前进一步的坐标变化

边界处理：

- 题目保证 `instructions.length >= 1`
- 指令只包含 `'G'`、`'L'`、`'R'`，所以最后的 `else` 可以视作右转
- 左转用 `(dir + 3) % 4`，不要写成 `(dir - 1) % 4`，C++ 中负数取模容易出问题
- 不需要模拟无限次，也不需要记录所有走过的位置

## 复杂度分析

- 时间复杂度：O(n)，其中 `n` 是指令长度，只执行一遍指令
- 空间复杂度：O(1)，只维护方向、坐标和固定方向数组

## 面试讲法

可以这样讲：

我先只模拟一遍指令，得到一轮结束后机器人的位置和方向。如果一轮后回到原点，那么它之后会重复这个闭环，所以一定有界。如果一轮后没有回到原点，但方向不再朝北，那么下一轮的位移方向会随朝向旋转，最多四轮后这些位移会互相抵消，也会形成环。只有当一轮后没回到原点且仍然朝北时，每一轮都会向同一个方向累加位移，机器人会无限远离原点。

可能追问：

- 为什么方向改变就一定有界：四个方向最多旋转 4 次回到北，四轮的位移向量会按方向旋转一圈，最终抵消
- 为什么不用记录路径：题目只问是否有界，不问具体环上有哪些点
- 为什么只执行一轮：一轮后的“位移 + 朝向变化”已经决定重复执行时的整体趋势
- `dx/dy` 怎么对应坐标：北是 `y + 1`，东是 `x + 1`，南是 `y - 1`，西是 `x - 1`

## 易错点

- 只判断是否回到原点，会漏掉 `GL` 这种方向改变但单轮未回原点的有界情况
- 左转不要写成 `(dir - 1) % 4`
- `dx/dy` 的顺序必须和 `dir` 编号保持一致
- 不要误以为必须走过相同坐标才算有环，本题判断的是“是否永远无法离开某个圆”
- 一轮后没回原点且仍朝北，才是无界

## 复盘卡片

- 一句话记忆：机器人有界 = 一轮后回到原点，或一轮后方向改变

- 如果写错，优先检查：左转取模、方向数组顺序、返回条件里的 `dir != 0`
