# 72. 编辑距离

## 题目信息

- 难度：中等
- 链接：https://leetcode.cn/problems/edit-distance/
- 主知识点：动态规划
- 相关知识点：字符串、二维 DP、序列匹配
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 给两个字符串，要求把一个变成另一个的最少操作数
- 操作包括插入、删除、替换
- 两个字符串的前缀状态会互相依赖
- 典型二维 DP：`dp[i][j]` 表示两个前缀之间的编辑距离

## 解题思路

### 暴力思路

可以从两个字符串末尾开始递归比较：

- 字符相同，就都往前走
- 字符不同，就尝试插入、删除、替换三种操作，取最小值

但递归过程中会反复计算相同的前缀状态，比如 `word1[0...i]` 和 `word2[0...j]` 的编辑距离，因此需要动态规划保存这些状态。

### 优化关键

定义：

```text
dp[i][j] = word1 的前 i 个字符转换成 word2 的前 j 个字符所需的最少操作数
```

这里 `i` 和 `j` 表示长度，不是下标。因此比较当前最后一个字符时，要看：

```cpp
word1[i - 1] 和 word2[j - 1]
```

初始化：

- `dp[i][0] = i`：把 `word1` 前 `i` 个字符变成空串，需要删除 `i` 次
- `dp[0][j] = j`：把空串变成 `word2` 前 `j` 个字符，需要插入 `j` 次

状态转移：

- 如果最后一个字符相同，不需要额外操作：

```cpp
dp[i][j] = dp[i - 1][j - 1];
```

- 如果最后一个字符不同，取三种操作的最小值再加一：

```cpp
dp[i][j] = min({
    dp[i - 1][j],     // 删除 word1[i - 1]
    dp[i][j - 1],     // 插入 word2[j - 1]
    dp[i - 1][j - 1]  // 替换 word1[i - 1] 为 word2[j - 1]
}) + 1;
```

### 最终做法

1. 记录 `m = word1.size()`，`n = word2.size()`
2. 创建 `(m + 1) * (n + 1)` 的 DP 数组
3. 初始化第一列 `dp[i][0] = i`
4. 初始化第一行 `dp[0][j] = j`
5. 从 `i = 1` 到 `m`、`j = 1` 到 `n` 填表
6. 字符相同则继承左上角
7. 字符不同则从删除、插入、替换中取最小值加一
8. 返回 `dp[m][n]`

## C++ 代码讲解

```cpp
class Solution {
public:
    int minDistance(string word1, string word2) {
        int m = word1.size();
        int n = word2.size();

        // dp[i][j] 表示 word1 前 i 个字符变成 word2 前 j 个字符的最少操作数
        vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));

        // word2 为空时，只能删除 word1 的前 i 个字符
        for (int i = 0; i <= m; ++i) {
            dp[i][0] = i;
        }

        // word1 为空时，只能插入 word2 的前 j 个字符
        for (int j = 0; j <= n; ++j) {
            dp[0][j] = j;
        }

        for (int i = 1; i <= m; ++i) {
            for (int j = 1; j <= n; ++j) {
                if (word1[i - 1] == word2[j - 1]) {
                    // 当前字符相同，不需要新增操作
                    dp[i][j] = dp[i - 1][j - 1];
                }else {
                    // 分别对应删除、插入、替换，选择代价最小的一种
                    dp[i][j] = min({dp[i - 1][j], dp[i][j - 1],dp[i - 1][j - 1]}) + 1;
                }
            }
        }

        return dp[m][n];
    }
};
```

关键点：

- `dp[i][j]`：两个前缀之间的最小编辑距离
- `dp[i][0] = i`：删除 `i` 个字符
- `dp[0][j] = j`：插入 `j` 个字符
- `word1[i - 1] == word2[j - 1]`：当前最后一个字符相同
- `dp[i - 1][j] + 1`：删除
- `dp[i][j - 1] + 1`：插入
- `dp[i - 1][j - 1] + 1`：替换

循环不变量 / 状态含义：

- 填到 `dp[i][j]` 时，它依赖的 `dp[i - 1][j]`、`dp[i][j - 1]`、`dp[i - 1][j - 1]` 都已经计算完成
- 每个状态只处理两个前缀之间的最优答案
- 最终 `dp[m][n]` 就是完整字符串之间的编辑距离

边界处理：

- `word1` 可以为空，此时答案是 `word2.size()`
- `word2` 可以为空，此时答案是 `word1.size()`
- `dp` 开成 `m + 1` 和 `n + 1`，是为了自然表示空前缀
- `i/j` 表示长度，访问字符时必须用 `i - 1` 和 `j - 1`

## 复杂度分析

- 时间复杂度：O(m * n)，每个状态计算一次
- 空间复杂度：O(m * n)，使用二维 DP 表

## 面试讲法

可以这样讲：

我定义 `dp[i][j]` 为 `word1` 的前 `i` 个字符变成 `word2` 的前 `j` 个字符需要的最少操作数。空串边界很好初始化：一个字符串变成空串需要全部删除，空串变成另一个字符串需要全部插入。转移时比较两个前缀的最后一个字符。如果相同，就不需要新增操作，直接继承 `dp[i-1][j-1]`；如果不同，就在删除、插入、替换三种操作里取最小值加一。最后返回 `dp[m][n]`。

可能追问：

- 为什么要开 `m + 1`、`n + 1`：为了包含空前缀状态
- 插入为什么对应 `dp[i][j - 1]`：先把 `word1` 前 `i` 个变成 `word2` 前 `j - 1` 个，再插入 `word2[j - 1]`
- 删除为什么对应 `dp[i - 1][j]`：先删除 `word1[i - 1]`，剩下前 `i - 1` 个去匹配前 `j` 个
- 能不能空间优化：可以压缩成一维，但二维表更适合第一次讲清楚状态

## 易错点

- `dp[i][j]` 里的 `i/j` 是前缀长度，不是字符下标
- 比较字符时要用 `word1[i - 1]` 和 `word2[j - 1]`
- 第一行和第一列必须初始化
- 字符相同时不要加一
- 字符不同时三种操作都要考虑：插入、删除、替换
