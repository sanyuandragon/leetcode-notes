# 200. 岛屿数量

## 题目信息

- 难度：中等
- 链接：https://leetcode.cn/problems/number-of-islands/
- 主知识点：图论
- 相关知识点：BFS、DFS、网格图、连通块
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 二维网格里有 `'1'` 和 `'0'`
- 四方向相邻的 `'1'` 属于同一个岛
- 要统计岛屿数量，本质是统计连通块数量
- 可以把每个格子看成图上的点，上下左右相邻就是边

## 解题思路

### 暴力思路

可以从每个陆地格子出发，反复寻找与它相连的所有陆地，但如果不做访问标记，同一座岛会被重复遍历很多次，也会重复计数。

### 优化关键

遍历整个网格。每当遇到一个还没访问过的陆地 `'1'`，说明发现了一座新的岛：

```cpp
ans++;
```

然后从这个格子开始 BFS，把这座岛上所有能四方向连通到的 `'1'` 都改成 `'0'`。这样后续扫描到这些格子时，就不会再重复计数。

这里把原网格当作访问数组使用：

- `'1'`：还没访问过的陆地
- `'0'`：水，或者已经访问过的陆地

### 最终做法

1. 记录行数 `m` 和列数 `n`
2. 准备四方向数组 `dx/dy`
3. 双重循环遍历每个格子
4. 如果当前格子是 `'1'`，岛屿数量 `ans++`
5. 把当前格子入队，并立刻标记为 `'0'`
6. BFS 队列不断弹出格子，检查四个方向
7. 如果邻居合法且是 `'1'`，入队并标记为 `'0'`
8. 扫描结束后返回 `ans`

## C++ 代码讲解

```cpp
class Solution {
public:
    int numIslands(vector<vector<char>>& grid) {
        int m = grid.size();    // row
        int n = grid[0].size(); // col
        int ans = 0;            // answer

        int dx[4] = {-1, 1, 0, 0};
        int dy[4] = {0, 0, -1, 1};

        for (int i = 0; i < m; ++i) {
            for (int j = 0; j < n; ++j) {
                if (grid[i][j] == '1') {
                    ans++;
                    queue<pair<int, int>> q;
                    q.push({i, j});
                    grid[i][j] = '0';

                    while (!q.empty()) {
                        pair<int, int> cur = q.front();
                        q.pop();
                        int x = cur.first;
                        int y = cur.second;

                        for (int k = 0; k < 4; ++k) {
                            int nx = x + dx[k];
                            int ny = y + dy[k];

                            if (nx >= 0 && nx < m && ny >= 0 && ny < n &&
                                grid[nx][ny] == '1') {
                                q.push({nx, ny});
                                grid[nx][ny] = '0';
                            }
                        }
                    }
                }
            }
        }

        return ans;
    }
};
```

关键点：

- `m`、`n`：网格行数和列数
- `ans`：已经发现的岛屿数量
- `dx/dy`：四方向移动数组，上、下、左、右
- `grid[i][j] == '1'`：发现一个尚未访问的新岛屿入口
- `grid[i][j] = '0'`：把访问过的陆地沉掉，防止重复访问
- `queue<pair<int, int>> q`：BFS 队列，保存待扩展的陆地格子
- `nx/ny`：当前格子的相邻位置

循环不变量 / 状态含义：

- 外层扫描过程中，所有已经处理过的岛都被改成了 `'0'`
- 每次遇到 `'1'`，它一定属于一座还没计数的新岛
- BFS 结束后，从起点连通到的整座岛都已经被标记为 `'0'`
- `ans` 始终等于目前已经发现并清理完的岛屿数量

边界处理：

- 题目保证 `1 <= m, n <= 300`，所以 `grid[0].size()` 是安全的
- 访问邻居前必须先判断 `nx/ny` 是否越界
- 入队时就要立刻标记为 `'0'`，不要等出队再标记，否则同一个格子可能被多个方向重复入队
- 注意字符是 `'1'` 和 `'0'`，不是整数 `1` 和 `0`

## 复杂度分析

- 时间复杂度：O(m * n)，每个格子最多入队并处理一次
- 空间复杂度：O(m * n)，最坏情况下队列可能存下大量陆地格子

## 面试讲法

可以这样讲：

我把二维网格看成图，每个陆地格子是一个节点，四方向相邻的陆地之间有边。题目要统计岛屿数量，本质就是统计连通块数量。我遍历所有格子，每遇到一个还没访问过的 `'1'`，就说明发现了一个新的连通块，答案加一。然后从这个点开始 BFS，把整座岛所有连通的陆地都标记为 `'0'`，表示已经访问过。这样后面扫描时不会重复计数。

可能追问：

- 为什么入队时就标记：避免同一个格子被多个邻居重复加入队列
- 能不能用 DFS：可以，递归或栈都行；大网格时递归 DFS 可能有栈深风险
- 能不能用并查集：可以，把相邻陆地合并，最后统计连通分量
- 为什么是四方向不是八方向：题目明确只允许水平或垂直相邻

## 易错点

- 忘记把起点 `grid[i][j]` 标记为 `'0'`
- 等出队后才标记，可能导致重复入队
- 判断边界时行列变量不要混：`nx` 对应 `m`，`ny` 对应 `n`
- 题目是字符网格，要比较 `grid[x][y] == '1'`
- 只统计岛屿入口，不能在 BFS 中每遇到一块陆地都 `ans++`
