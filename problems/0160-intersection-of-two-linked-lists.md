# 160. 相交链表

## 题目信息

- 难度：简单
- 链接：https://leetcode.cn/problems/intersection-of-two-linked-lists/
- 主知识点：链表
- 相关知识点：双指针、指针切换、长度差
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 给两个单链表头节点 `headA` 和 `headB`
- 要找两个链表相交的起始节点
- 相交指的是两个指针指向同一个节点地址，不是节点值相等
- 要求时间 O(m + n)、空间 O(1)，并且不能破坏原链表结构

## 解题思路

### 暴力思路

可以用哈希集合存下链表 A 的所有节点地址，再遍历链表 B，第一次遇到已经出现过的节点就是交点。这个方法时间是 O(m + n)，但需要 O(m) 额外空间。

也可以分别求两个链表长度，让长链表先走长度差，再一起走。这是 O(1) 空间的标准做法。

### 优化关键

双指针切换链表可以自动抵消长度差：

- 指针 `p` 从链表 A 开始走，走到空后切到链表 B
- 指针 `q` 从链表 B 开始走，走到空后切到链表 A

如果两个链表相交，`p` 和 `q` 最终会在第一个相交节点相遇。因为它们都走了：

```text
A 的非公共部分 + B 的非公共部分 + 公共部分
```

如果两个链表不相交，两个指针最终都会走到 `nullptr`，此时 `p == q`，循环结束并返回空。

### 最终做法

1. 初始化 `p = headA`，`q = headB`
2. 当 `p != q` 时循环
3. `p` 不为空就走向 `p->next`，为空就切到 `headB`
4. `q` 不为空就走向 `q->next`，为空就切到 `headA`
5. 循环结束时，`p` 就是交点；如果没有交点，则是 `nullptr`

## C++ 代码讲解

```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode(int x) : val(x), next(NULL) {}
 * };
 */
class Solution {
public:
    ListNode *getIntersectionNode(ListNode *headA, ListNode *headB) {
        ListNode* p = headA;
        ListNode* q = headB;

        // 两个指针走过 A+B 和 B+A，总路程相同；相交则在交点相遇，不相交则同时到 nullptr
        while (p != q) {
            if (p != nullptr) {
                p = p->next;
            }else {
                p = headB;
            }

            if (q != nullptr) {
                q = q->next;
            }else {
                q = headA;
            }
        }

        return p;
    }
};
```

关键点：

- `p`：从链表 A 出发，走完 A 后切到 B
- `q`：从链表 B 出发，走完 B 后切到 A
- `p != q`：比较的是节点地址，不是节点值
- `p = headB`：让从 A 出发的指针补走 B 的长度
- `q = headA`：让从 B 出发的指针补走 A 的长度
- `return p`：循环结束时，`p` 可能是交点，也可能是 `nullptr`

循环不变量 / 状态含义：

- `p` 和 `q` 每轮都前进一步
- `p` 的完整路径是 `A -> B`
- `q` 的完整路径是 `B -> A`
- 两个指针最多各走 `m + n` 步，就会在交点或 `nullptr` 相遇

边界处理：

- 如果 `headA == nullptr` 或 `headB == nullptr`，两个指针最终都会变成 `nullptr`
- 如果两个链表一开始就相交，即 `headA == headB`，循环不会进入，直接返回头节点
- 不需要修改任何 `next` 指针，满足题目要求的“保持原结构”
- 相交节点之后的所有节点是同一段共享链表

## 复杂度分析

- 时间复杂度：O(m + n)，两个指针最多各走完两个链表
- 空间复杂度：O(1)，只使用两个指针

## 面试讲法

可以这样讲：

我用两个指针分别从两个链表头出发。指针走到自己链表末尾后，就切换到另一个链表的头继续走。这样两个指针最终走过的总长度都是 `m + n`，长度差会被自动抵消。如果两个链表相交，它们会在第一个公共节点相遇；如果不相交，它们会同时走到 `nullptr`。整个过程不修改链表结构，空间复杂度是 O(1)。

可能追问：

- 为什么能抵消长度差：一个指针先走 A 再走 B，另一个先走 B 再走 A，总路程相同
- 为什么比较指针不是比较值：相交表示两个链表共享同一个节点对象，值相同不代表相交
- 不相交会不会死循环：不会，两个指针最多各走 `m + n` 步后同时为 `nullptr`
- 能不能先求长度差：可以，先让长链表走差值，再同步走，也能 O(1) 空间完成

## 模板沉淀

- 是否沉淀模板：是

- 可复用点：两个链表存在长度差时，可以让两个指针交换起点，使总路程一致

## 易错点

- 相交判断必须比较节点地址：`p == q`
- 不要比较 `p->val == q->val`
- 指针走到 `nullptr` 后才切换到另一个链表头
- 不要修改链表结构，比如把尾节点接到另一个链表上
- 返回值可能是 `nullptr`，表示两个链表不相交

## 复盘卡片

- 一句话记忆：相交链表 = A 指针走 A+B，B 指针走 B+A，路程相同后在交点或空处相遇
- 下次先想：相交是节点地址相同，不是值相同
- 如果写错，优先检查：是否比较指针、空后是否切换链表头、有没有破坏原链表
