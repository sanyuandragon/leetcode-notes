# 236. 二叉树的最近公共祖先

## 题目信息

- 难度：中等
- 链接：https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/
- 主知识点：二叉树
- 相关知识点：DFS、递归、后序思想、最近公共祖先
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 输入是普通二叉树，不是二叉搜索树
- 要找两个节点 `p` 和 `q` 的最近公共祖先
- 一个节点可以是它自己的祖先
- 需要根据左右子树里是否找到目标节点来决定答案

## 解题思路

### 暴力思路

可以先从根节点 DFS，分别记录 `p` 和 `q` 到根节点的路径，然后比较两条路径，最后一个相同节点就是最近公共祖先。

这个做法直观，但需要额外保存两条路径，代码也会更长。

### 优化关键

递归函数可以设计成：

> 在以 `root` 为根的子树中，寻找 `p` 和 `q`，并返回当前子树中已经能确定的目标节点或最近公共祖先。

递归到某个节点时：

- 如果 `root == nullptr`，说明这棵子树里没找到，返回空
- 如果 `root == p` 或 `root == q`，说明找到目标节点，直接返回 `root`
- 递归去左右子树找
- 如果左右子树都返回非空，说明 `p` 和 `q` 分别在两边，当前 `root` 就是最近公共祖先
- 如果只有一边非空，把那一边的结果继续往上返回

### 最终做法

1. 如果当前节点为空，或者当前节点就是 `p/q`，直接返回当前节点
2. 递归搜索左子树，得到 `left`
3. 递归搜索右子树，得到 `right`
4. 如果 `left` 和 `right` 都非空，返回当前 `root`
5. 否则返回非空的一边

## C++ 代码讲解

```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode(int x) : val(x), left(NULL), right(NULL) {}
 * };
 */
class Solution {
public:
    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        // 空节点表示没找到；当前节点等于 p/q 时，当前节点就是可向上返回的目标
        if (root == nullptr || root == p || root == q) {
            return root;
        }

        // 分别在左右子树中寻找 p 和 q
        TreeNode* left = lowestCommonAncestor(root->left, p, q);
        TreeNode* right = lowestCommonAncestor(root->right, p, q);

        // 左右子树都找到目标，说明 p 和 q 分居两侧，当前 root 就是最近公共祖先
        if (left && right) {
            return root;
        }

        // 只有一侧找到时，把找到的节点或祖先继续往上返回
        return left ? left : right;
    }
};
```

关键点：

- `root == nullptr`：当前子树没有找到目标节点
- `root == p || root == q`：当前节点就是目标之一，可以直接向上返回
- `left`：左子树中找到的目标节点或最近公共祖先
- `right`：右子树中找到的目标节点或最近公共祖先
- `left && right`：两个目标分别出现在左右子树，当前节点就是分叉点
- `left ? left : right`：只有一边找到，就把这一边的结果继续返回给父节点

循环不变量 / 状态含义：

- 对任意节点 `root`，函数返回值表示当前子树中能找到的 `p/q` 或它们的最近公共祖先
- 如果当前子树中一个目标都没有，返回 `nullptr`
- 如果当前子树中只找到一个目标，返回那个目标节点
- 如果当前子树中两个目标都找到了，返回它们在当前子树内的最近公共祖先

边界处理：

- 题目保证 `p` 和 `q` 都存在于二叉树中，所以不需要额外处理只存在一个节点的情况
- `p != q`，不用考虑两个目标是同一个节点
- “一个节点可以是它自己的祖先”，所以遇到 `root == p` 或 `root == q` 可以立刻返回
- 这是普通二叉树，不能使用二叉搜索树的大小关系来判断方向

## 复杂度分析

- 时间复杂度：O(n)，最坏情况下需要访问每个节点一次
- 空间复杂度：O(h)，其中 `h` 是二叉树高度，递归调用栈最坏为 O(n)

## 面试讲法

可以这样讲：

我用 DFS 递归处理。当前函数的返回值定义为：在当前子树中找到的目标节点，或者已经确定的最近公共祖先。递归时如果遇到空节点就返回空；如果当前节点是 `p` 或 `q`，就直接返回当前节点。然后分别递归左右子树。如果左右子树都返回非空，说明 `p` 和 `q` 分别在当前节点两侧，所以当前节点就是最近公共祖先。否则说明两个节点都在同一侧，或者只找到一个目标，就把非空结果继续往上返回。

可能追问：

- 为什么遇到 `p/q` 可以直接返回：因为节点本身也可以作为自己的祖先，若另一个节点在它的子树里，最终这个节点就是答案
- 为什么左右都非空时返回当前节点：当前节点是第一次把两个目标分到左右两侧的节点，也就是最近公共祖先
- 如果是二叉搜索树怎么办：可以利用大小关系往左或往右走，那是另一题
- 如果题目不保证 `p/q` 存在怎么办：需要额外统计是否两个节点都被找到

## 易错点

- 忘记“节点可以是自己的祖先”，导致 `root == p/q` 时没有直接返回
- 把普通二叉树误当成二叉搜索树，用节点值大小判断方向
- 不清楚递归函数返回值含义，导致 `left/right` 的判断写乱
- 左右都非空时返回的是当前 `root`，不是 `left` 或 `right`
- 题目比较的是节点指针 `root == p`，不是比较节点值
