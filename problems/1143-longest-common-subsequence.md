# 1143. 最长公共子序列

## 题目信息

- 难度：中等
- 链接：https://leetcode.cn/problems/longest-common-subsequence/
- 主知识点：动态规划
- 相关知识点：字符串、二维 DP、子序列
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 给两个字符串，要求最长公共子序列长度
- 子序列不要求连续，但必须保持原字符串相对顺序
- 两个字符串同时参与匹配，适合二维 DP
- `dp[i][j]` 通常表示两个前缀之间的最优答案

## 解题思路

### 暴力思路

可以枚举 `text1` 的所有子序列，再判断它是否也是 `text2` 的子序列，并取最大长度。但一个长度为 `m` 的字符串有 `2^m` 个子序列，显然不可行。

### 优化关键

定义：

```text
dp[i][j] = text1 的前 i 个字符和 text2 的前 j 个字符的最长公共子序列长度
```

这里 `i` 和 `j` 是前缀长度，不是字符下标。因此比较当前最后一个字符时，要看：

```cpp
text1[i - 1] 和 text2[j - 1]
```

如果两个字符相同，它们可以作为公共子序列的最后一个字符：

```cpp
dp[i][j] = dp[i - 1][j - 1] + 1;
```

如果两个字符不同，当前最后两个字符不能同时匹配，只能尝试丢掉其中一个：

```cpp
dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
```

### 最终做法

1. 记录 `m = text1.size()`，`n = text2.size()`
2. 创建 `(m + 1) * (n + 1)` 的 DP 数组，初始为 0
3. 从 `i = 1` 到 `m` 遍历 `text1` 前缀
4. 从 `j = 1` 到 `n` 遍历 `text2` 前缀
5. 如果当前字符相同，继承左上角并加一
6. 如果当前字符不同，取上方和左方的最大值
7. 返回 `dp[m][n]`

## C++ 代码讲解

```cpp
class Solution {
public:
    int longestCommonSubsequence(string text1, string text2) {
        int m = text1.size();
        int n = text2.size();

        // dp[i][j] 表示 text1 前 i 个字符和 text2 前 j 个字符的 LCS 长度
        vector<vector<int>> dp(m + 1, vector<int> (n + 1, 0));

        for (int i = 1; i <= m; ++i) {
            for (int j = 1; j <= n; ++j) {
                if (text1[i - 1] == text2[j - 1]) {
                    // 当前两个字符相同，可以接在前面公共子序列后面
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                }else {
                    // 当前字符不同，只能分别尝试跳过 text1 或 text2 的最后一个字符
                    dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
                }
            }
        }

        return dp[m][n];
    }
};
```

关键点：

- `dp[i][j]`：两个前缀的最长公共子序列长度
- `m + 1`、`n + 1`：多出来的一行一列表示空前缀
- `text1[i - 1] == text2[j - 1]`：两个前缀最后一个字符相同
- `dp[i - 1][j - 1] + 1`：把当前相同字符接到 LCS 后面
- `max(dp[i - 1][j], dp[i][j - 1])`：字符不同，跳过其中一个末尾字符

循环不变量 / 状态含义：

- 填到 `dp[i][j]` 时，依赖的左上、上方、左方状态都已经计算完成
- 第一行和第一列是空字符串参与匹配，公共子序列长度为 0
- 每个状态只关心两个前缀之间的最优 LCS 长度

边界处理：

- 如果任意一个字符串为空，答案是 0
- `dp` 初始为 0，天然覆盖第一行和第一列
- `i/j` 是长度，访问字符时要用 `i - 1` 和 `j - 1`
- 子序列不要求连续，不能当成子串处理

## 复杂度分析

- 时间复杂度：O(m * n)，每个状态计算一次
- 空间复杂度：O(m * n)，使用二维 DP 表

## 面试讲法

可以这样讲：

我定义 `dp[i][j]` 表示 `text1` 前 `i` 个字符和 `text2` 前 `j` 个字符的最长公共子序列长度。空前缀和任何字符串的 LCS 都是 0。填表时比较两个前缀的最后一个字符。如果相同，它可以作为公共子序列的最后一个字符，所以 `dp[i][j] = dp[i-1][j-1] + 1`。如果不同，就说明这两个末尾字符不能同时用，只能跳过其中一个，因此取 `dp[i-1][j]` 和 `dp[i][j-1]` 的最大值。最后答案是 `dp[m][n]`。

可能追问：

- 为什么字符不同只看上方和左方：分别表示跳过 `text1` 当前字符或跳过 `text2` 当前字符
- 为什么不需要初始化第一行列：默认 0 就代表空字符串和任意前缀的 LCS 长度
- 子序列和子串区别：子序列可以不连续，子串必须连续
- 能不能空间优化：可以压缩成一维，但二维表更容易讲清状态

## 易错点

- `i/j` 表示前缀长度，不是下标
- 比较字符时用 `text1[i - 1]` 和 `text2[j - 1]`
- 字符相同时是 `dp[i - 1][j - 1] + 1`
- 字符不同时取 `max(dp[i - 1][j], dp[i][j - 1])`
- 不要把最长公共子序列写成最长公共子串

## 复盘卡片

- 一句话记忆：LCS = 两个前缀 DP，相等左上加一，不等取上左最大
- 下次先想：子序列可以跳过字符，所以字符不同就是跳过其中一个末尾
- 如果写错，优先检查：空前缀、`i - 1/j - 1`、相等和不等时的转移
