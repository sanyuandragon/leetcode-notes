# 3. 无重复字符的最长子串

## 题目信息

- 难度：中等
- 链接：https://leetcode.cn/problems/longest-substring-without-repeating-characters/
- 主知识点：滑动窗口
- 相关知识点：字符串、哈希集合、双指针
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 要找的是连续子串，不是子序列
- 要求窗口内字符不能重复
- 需要找最长长度
- 左右边界都只会向右移动，适合滑动窗口

## 解题思路

### 暴力思路

可以枚举所有子串，再判断每个子串是否有重复字符。子串数量是 O(n^2)，每个子串判断最坏还要 O(n)，复杂度太高。

### 优化关键

维护一个窗口 `[left, right]`，保证窗口内没有重复字符。用哈希集合 `window` 记录当前窗口中的字符。

对于每个 `left`：

- 如果 `left` 右移了，就把移出窗口的 `s[left - 1]` 从集合中删除
- 然后不断尝试让 `right` 右移
- 只要 `s[right + 1]` 不在窗口中，就可以加入窗口
- 当右侧下一个字符会重复时，当前窗口已经是以 `left` 为左边界的最长合法窗口

每次更新答案：

```cpp
ans = max(ans, right - left + 1);
```

### 最终做法

1. 创建哈希集合 `window`
2. 初始化 `right = -1`，表示窗口一开始为空
3. 枚举左边界 `left`
4. 如果 `left != 0`，删除上一轮移出窗口的字符
5. 不断扩张右边界，直到下一个字符会重复或越界
6. 用当前窗口长度更新答案
7. 返回最大长度

## C++ 代码讲解

```cpp
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        unordered_set<int> window;
        int n = s.size();
        int right = -1;
        int ans = 0;

        for (int left = 0; left < n; ++left) {
            if (left != 0) {
                window.erase(s[left - 1]);
            }

            while (right + 1 < n && !window.count(s[right + 1])) {
                window.insert(s[right + 1]);
                ++right;
            }

            ans = max(ans, right - left + 1);
        }

        return ans;
    }
};
```

关键点：

- `window`：保存当前窗口内已有字符，用来 O(1) 判断重复
- `right = -1`：窗口初始为空，方便第一次加入 `s[0]`
- `left`：窗口左边界，每轮向右移动一格
- `window.erase(s[left - 1])`：左边界右移后，移除离开窗口的字符
- `right + 1 < n`：保证访问 `s[right + 1]` 不越界
- `!window.count(s[right + 1])`：下一个字符不重复，才能继续扩张
- `right - left + 1`：当前合法窗口长度

循环不变量 / 状态含义：

- 每轮更新答案前，窗口 `[left, right]` 内没有重复字符
- `right` 对当前 `left` 已经尽量向右扩张，下一位要么越界，要么会重复
- `window` 中的字符与当前窗口内容保持一致
- `left` 和 `right` 都只向右移动，因此整体是线性复杂度

边界处理：

- 空字符串时，`n = 0`，循环不进入，返回 `0`
- `right` 初始化为 `-1`，空窗口长度在第一次循环前不会被直接使用
- 当窗口被收缩到空时，`right` 可能小于 `left`，长度公式仍然可在扩张后正确工作
- 题目字符包含字母、数字、符号和空格，使用哈希集合比固定 26 长度数组更通用

## 复杂度分析

- 时间复杂度：O(n)，每个字符最多被右指针加入一次、被左指针移除一次
- 空间复杂度：O(min(n, C))，`C` 是字符集大小

## 面试讲法

可以这样讲：

这题要求最长的不含重复字符的连续子串，所以我用滑动窗口维护一个始终不含重复字符的区间。窗口右端尽量向右扩展，只要下一个字符没有出现在窗口里就加入；如果下一个字符会重复，就说明当前左边界下已经不能再扩了，更新答案。然后左边界右移，并从集合中移除离开的字符。因为左右指针都只向右移动，所以时间复杂度是 O(n)。

可能追问：

- 为什么不是子序列：题目要求子串，必须连续
- 为什么能用滑动窗口：窗口合法性可以通过移动左边界恢复，左右边界都是单调右移
- 能不能用数组代替 `unordered_set`：如果字符集固定，可以用数组记录出现位置或计数
- 如何写成更常见的 `right` for 循环：右指针每次加入字符，若重复就不断移动左指针直到合法

## 易错点

- 题目要的是子串，不是子序列；`"pwwkew"` 答案是 `"wke"`，不是 `"pwke"`
- 左指针移动时，必须从窗口中删除离开的字符
- 右指针访问 `right + 1` 前要判断越界
- 窗口长度是 `right - left + 1`
- `unordered_set<int>` 能用，但语义上写 `unordered_set<char>` 更贴合字符集合

## 复盘卡片

- 一句话记忆：无重复最长子串 = 窗口内不重复，右端尽量扩，左端移除旧字符
- 下次先想：窗口里要维护哪些状态；这里就是当前字符集合
- 如果写错，优先检查：左端移除、右端越界、答案长度公式
