# 141. 环形链表

## 题目信息

- 难度：简单
- 链接：https://leetcode.cn/problems/linked-list-cycle/
- 主知识点：链表
- 相关知识点：快慢指针、双指针、Floyd 判环
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 题目要求判断链表中是否有环
- 不能只靠遍历到 `nullptr`，因为有环时会无限走下去
- 要求 O(1) 内存时，优先想到快慢指针
- 快指针一次走两步，慢指针一次走一步，有环就会相遇

## 解题思路

### 暴力思路

可以用哈希集合记录访问过的节点地址。如果再次遇到同一个节点，说明有环；如果走到 `nullptr`，说明无环。这个方法时间是 O(n)，但需要 O(n) 额外空间。

### 优化关键

用快慢指针可以把空间降到 O(1)：

- `slow` 每次走一步
- `fast` 每次走两步

如果链表无环，`fast` 会先走到 `nullptr` 或 `fast->next == nullptr`。

如果链表有环，`fast` 和 `slow` 进入环后，`fast` 每轮都会比 `slow` 多走一步，相当于在环上追赶，最终一定会相遇。

### 最终做法

1. 初始化 `slow = head`，`fast = head`
2. 当 `fast != nullptr && fast->next != nullptr` 时循环
3. `fast` 走两步，`slow` 走一步
4. 如果二者相等，说明有环，返回 `true`
5. 如果循环结束，说明 `fast` 到达链表尾部，返回 `false`

## C++ 代码讲解

```cpp
class Solution {
public:
    bool hasCycle(ListNode *head) {
        ListNode* slow = head;
        ListNode* fast = head;

        while (fast != nullptr && fast->next != nullptr) {
            fast = fast->next->next;
            slow = slow->next;

            if (fast == slow) {
                return true;
            }
        }

        return false;
    }
};
```

关键点：

- `slow`：慢指针，每轮走一步
- `fast`：快指针，每轮走两步
- `fast != nullptr && fast->next != nullptr`：保证 `fast->next->next` 安全
- `fast == slow`：比较的是节点地址是否相同，不是节点值是否相同
- 循环结束后返回 `false`：说明快指针能走到链表尾部，没有环

循环不变量 / 状态含义：

- 每轮循环开始时，如果还能继续，`fast` 至少可以安全走两步
- 如果链表无环，`fast` 会比 `slow` 更早到达尾部
- 如果链表有环，两个指针进入环后，快指针会逐步缩短与慢指针的距离，直到相遇

边界处理：

- 空链表：`head == nullptr`，循环不进入，返回 `false`
- 单节点无环：`fast->next == nullptr`，循环不进入，返回 `false`
- 单节点自环：`fast->next != nullptr`，循环进入后会相遇，返回 `true`
- 判断循环条件时必须同时检查 `fast` 和 `fast->next`

## 复杂度分析

- 时间复杂度：O(n)，无环时最多走到尾部，有环时进入环后有限步内相遇
- 空间复杂度：O(1)，只使用两个指针

## 面试讲法

可以这样讲：

我用快慢指针判断链表是否有环。慢指针每次走一步，快指针每次走两步。如果链表没有环，快指针会先走到空指针，说明能正常到达链表尾部。如果链表有环，两个指针进入环之后，快指针每轮比慢指针多走一步，相当于在环里追赶慢指针，最终一定会相遇。相遇时返回 true，否则返回 false。

可能追问：

- 为什么有环一定会相遇：进入环后，快慢指针的相对距离每轮减少 1，距离在环长范围内循环，最终会变成 0
- 为什么比较指针而不是值：不同节点可能有相同值，环判断要看是否回到同一个节点
- 能不能用哈希表：可以，记录访问过的节点地址，空间 O(n)
- 如果要找入环点怎么办：相遇后让一个指针回到 `head`，两个指针每次走一步，再次相遇处就是入环点

## 易错点

- 循环条件必须是 `fast != nullptr && fast->next != nullptr`
- `fast = fast->next->next` 前不能少判空
- 判断相遇要比较指针地址：`fast == slow`
- 不要比较 `fast->val == slow->val`
- `pos` 只是 LeetCode 用来构造环的内部参数，不会传给函数

## 复盘卡片

- 一句话记忆：链表判环 = 快慢指针，快的如果追上慢的就有环，否则快的先到空
