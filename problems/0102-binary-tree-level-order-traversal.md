# 102. 二叉树的层序遍历

## 题目信息

- 难度：中等
- 链接：https://leetcode.cn/problems/binary-tree-level-order-traversal/
- 主知识点：二叉树
- 相关知识点：BFS、队列、层序遍历
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 题目要求“逐层地，从左到右”访问节点
- 返回值是 `vector<vector<int>>`，每一层单独放一个数组
- 二叉树层序遍历天然对应 BFS
- 需要用队列维护下一批要访问的节点

## 解题思路

### 暴力思路

可以用 DFS 递归并记录深度，把每个节点放到对应深度的数组里。这也能做，但题目直接要求层序遍历，用 BFS 队列更贴近“按层访问”的过程。

### 优化关键

BFS 的核心是队列。先把根节点入队，每次从队头取出一个节点，再把它的左右孩子入队。

但本题不是返回一个扁平数组，而是每一层单独返回一个数组。因此每轮处理前，要先记录：

```cpp
int size = q.size();
```

这个 `size` 表示当前层的节点数量。接下来只处理这 `size` 个节点。处理过程中入队的孩子属于下一层，不能混进当前层。

### 最终做法

1. 如果 `root == nullptr`，直接返回空数组
2. 创建队列 `q`，把 `root` 放进去
3. 当队列不为空时，先记录当前层节点数 `size`
4. 创建当前层数组 `level`
5. 循环处理 `size` 个节点
6. 每弹出一个节点，就记录它的值，并把非空左右孩子入队
7. 当前层处理完后，把 `level` 放进答案
8. 返回答案

## C++ 代码讲解

```cpp
class Solution {
public:
    vector<vector<int>> levelOrder(TreeNode* root) {
        vector<vector<int>> ans;

        if (root == nullptr) {
            return ans;
        }

        queue<TreeNode*> q;
        q.push(root);

        while (!q.empty()) {
            int size = q.size();
            vector<int> level;

            for (int i = 0; i < size; ++i) {
                TreeNode* node = q.front();
                q.pop();

                level.push_back(node->val);

                if (node->left != nullptr) {
                    q.push(node->left);
                }
                if (node->right != nullptr) {
                    q.push(node->right);
                }
            }

            ans.push_back(level);
        }

        return ans;
    }
};
```

关键点：

- `ans`：最终答案，每个元素代表一层
- `root == nullptr`：空树直接返回空数组
- `queue<TreeNode*> q`：BFS 队列，保存接下来要访问的节点
- `size = q.size()`：当前层的节点数量，是分层的关键
- `level`：当前层的所有节点值
- `q.push(node->left/right)`：孩子节点入队，等待下一层处理
- 先入左孩子再入右孩子，保证每层从左到右

循环不变量 / 状态含义：

- 每次进入 `while` 时，队列中已经按顺序保存了当前层的所有节点
- `size` 固定后，本轮 `for` 只处理当前层节点
- 在本轮 `for` 中新加入队列的节点都属于下一层
- 每轮结束时，`level` 正好包含当前层从左到右的节点值

边界处理：

- 空树必须提前返回，否则把空指针入队会出错
- `size` 要在每层开始时记录，不能在 `for` 循环条件中动态使用 `q.size()`
- 只有孩子节点非空时才入队，否则之后访问 `node->val` 会空指针崩溃
- 节点数范围允许为 `0`，所以空树是合法输入

## 复杂度分析

- 时间复杂度：O(n)，每个节点入队、出队各一次
- 空间复杂度：O(n)，最坏情况下队列保存一层的大量节点，答案本身也需要 O(n)

## 面试讲法

可以这样讲：

层序遍历本质上是 BFS。我用队列保存当前待访问的节点。为了让返回结果按层分组，每一轮开始时先记录队列长度，这个长度就是当前层的节点数。然后只弹出并处理这 `size` 个节点，过程中把它们的左右孩子加入队列。因为新加入的孩子都排在队列后面，所以它们会在下一轮作为下一层处理。空树时直接返回空数组。

可能追问：

- 为什么要先保存 `size`：否则循环过程中孩子入队会改变队列长度，当前层和下一层会混在一起
- 为什么用队列而不是栈：队列先进先出，符合从上到下、从左到右的 BFS 顺序
- 能不能用 DFS：可以，递归时带 `depth`，把节点值放到 `ans[depth]`
- 队列最大空间是多少：最坏是树最宽的一层，量级 O(n)

## 易错点

- 忘记判断 `root == nullptr`
- `for` 循环里不要直接写 `i < q.size()`，因为 `q.size()` 会随着孩子入队变化
- 当前层数组 `level` 要在每一层重新创建
- 左右孩子入队前要判空
- 需要按层输出时，不能只用普通 BFS 扁平遍历

## 复盘卡片

- 一句话记忆：层序遍历 = 队列 BFS，每轮先固定当前层 `size`
- 下次先想：返回是不是按层分组；如果是，必须用 `size` 分隔当前层和下一层
- 如果写错，优先检查：空树、`size` 是否提前保存、孩子是否判空入队
