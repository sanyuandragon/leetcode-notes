# 15. 三数之和

## 题目信息

- 难度：中等
- 链接：https://leetcode.cn/problems/3sum/
- 主知识点：双指针
- 相关知识点：排序、去重、数组
- 复习状态：`new`

## 一眼识别

看到这些信号时，应该想到本题方法：

- 要找三元组，且答案不能重复
- 只要求返回数值组合，不要求保留原下标
- 数组可以排序，排序后两数之和可以用双指针线性扫描
- 目标和固定为 `0`，枚举一个数后，剩下两个数转成 `two sum`

## 解题思路

### 暴力思路

最直接的做法是三重循环枚举 `i, j, k`，判断 `nums[i] + nums[j] + nums[k] == 0`。但 `n` 最大为 `3000`，三重循环是 O(n^3)，会超时。

### 优化关键

三元组不要求原始下标，只关心数值组合，因此可以先排序。排序后固定第一个数 `nums[i]`，问题变成在 `i + 1` 到 `n - 1` 的有序区间里找两个数，使它们的和等于 `-nums[i]`。

有序数组里的两数之和可以用对撞双指针：

- 如果当前三数之和小于 `0`，说明左指针对应的数太小，`l++`
- 如果当前三数之和大于 `0`，说明右指针对应的数太大，`r--`
- 如果等于 `0`，记录答案，然后同时移动左右指针，并跳过重复值

### 最终做法

1. 排序 `nums`
2. 枚举第一个数 `i`
3. 如果 `nums[i] > 0`，后面的数都不小于它，不可能再凑出 `0`，直接结束
4. 如果 `nums[i] == nums[i - 1]`，跳过，避免重复三元组
5. 在右侧区间用 `l`、`r` 对撞找两数之和
6. 找到答案后，移动 `l` 和 `r`，并跳过重复值

## C++ 代码讲解

```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        vector<vector<int>> res;
        sort(nums.begin(), nums.end());

        int n = nums.size();
        for (int i = 0; i < n; i++) {
            if (nums[i] > 0) {
                break;
            }
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }

            int l = i + 1;
            int r = n - 1;
            while (l < r) {
                int sum = nums[i] + nums[l] + nums[r];
                if (sum == 0) {
                    res.push_back({nums[i], nums[l], nums[r]});
                    l++;
                    r--;

                    while (l < r && nums[l] == nums[l - 1]) {
                        l++;
                    }
                    while (l < r && nums[r] == nums[r + 1]) {
                        r--;
                    }
                } else if (sum < 0) {
                    l++;
                } else {
                    r--;
                }
            }
        }

        return res;
    }
};
```

关键点：

- `sort(nums.begin(), nums.end())`：排序是去重和双指针成立的前提
- `i`：固定三元组的第一个数，把三数之和降成两数之和
- `l`、`r`：在 `i` 右侧区间寻找另外两个数，保证下标互不相同
- `nums[i] > 0`：排序后如果第一个数已经大于 `0`，三数之和必然大于 `0`
- `i > 0 && nums[i] == nums[i - 1]`：跳过重复的第一个数
- 找到答案后再跳过 `l`、`r` 的重复值，避免重复三元组

循环不变量 / 状态含义：

- 外层循环固定 `i`，只在 `i` 右侧寻找配对，因此不会重复使用同一个元素
- 内层循环中，`l` 和 `r` 始终满足 `i < l < r`
- 因为数组有序，`sum < 0` 时只能增大左侧值，`sum > 0` 时只能减小右侧值

边界处理：

- 输入长度至少为 `3`，但代码对更短数组也能自然返回空结果
- 去重时必须先移动 `l++`、`r--`，再和移动前的相邻值比较
- `while (l < r && nums[l] == nums[l - 1])` 中的 `l - 1` 是刚刚记录过的旧值
- `while (l < r && nums[r] == nums[r + 1])` 中的 `r + 1` 是刚刚记录过的旧值

## 复杂度分析

- 时间复杂度：O(n^2)。排序是 O(n log n)，外层枚举 O(n)，内层双指针总计 O(n)
- 空间复杂度：O(log n) 到 O(n)，取决于排序实现的递归栈；不计答案数组

## 面试讲法

可以这样讲：

这题如果直接枚举三个下标会是 O(n^3)。因为题目只要求不重复的数值三元组，不要求原下标，所以我先排序。排序后枚举第一个数 `nums[i]`，剩下的问题就是在右侧有序区间里找两个数，使三数之和为 `0`。这个两数查找可以用双指针：和小了移动左指针，和大了移动右指针。为了避免重复答案，固定第一个数时跳过重复值，找到一组三元组后也跳过左右指针上的重复值。

可能追问：

- 为什么排序不会影响答案：题目返回的是数值三元组，不要求原下标
- 为什么 `nums[i] > 0` 可以提前结束：排序后后续元素都不小于 `nums[i]`
- 为什么不会漏答案：对固定的 `i`，双指针根据有序性只丢弃不可能形成答案的一侧
- 如何去重：三个位置分层去重，`i` 去重负责避免相同开头，`l/r` 去重负责避免相同后两个数

## 模板沉淀

- 是否沉淀模板：是
- 对应模板：[排序后固定一个数 + 双指针](../templates/cpp/two-pointers.md)
- 可复用点：`k sum` 中常见的“排序 + 固定前缀 + 双指针收尾”思路

## 易错点

- 忘记对 `i` 去重，会产生重复三元组
- 找到答案后只移动指针但不跳过重复值，也会产生重复结果
- 去重循环必须带 `l < r`，否则可能越界或破坏指针关系
- `sum < 0` 时移动 `l`，`sum > 0` 时移动 `r`，方向不要反
- 如果题目要求原下标，排序会改变下标，需要额外保存原始位置；本题不需要

## 复盘卡片

- 一句话记忆：三数之和 = 排序后固定一个数，把剩下两个数交给对撞双指针
- 下次先想：题目是否只要数值组合；如果是，就可以排序并分层去重
- 如果写错，优先检查：`i` 去重、命中答案后的 `l/r` 去重、`sum` 判断后的指针移动方向
