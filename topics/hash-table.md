# 哈希表

## 核心识别

什么时候想到哈希表：

- 需要快速判断一个元素是否出现过
- 需要把值映射到下标、次数、分组或状态
- 暴力枚举里存在“查找另一个值”的步骤

## 常用套路

- `unordered_map<key, value>`：记录下标、次数、最近位置
- `unordered_set<T>`：只关心是否存在
- 边遍历边查询：先查答案，再插入当前元素，避免和自己配对
- 频次统计：先统计，再按条件消费或比较

## 易错点

- `unordered_map[key]` 会在 key 不存在时插入默认值；只判断存在性时优先用 `find`
- 需要返回下标时，确认存的是原数组下标，不是排序后的下标
- 边遍历边插入时，注意“先查后插”和“先插后查”的区别
- `insert` 不会覆盖已存在的 key，`mp[key] = value` 会覆盖；根据题意选择

## 面试表达

这类题的本质是用空间换时间，把原本 O(n) 的查找压到均摊 O(1)，从而减少一层枚举。
