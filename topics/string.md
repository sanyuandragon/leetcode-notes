# 字符串

## 核心识别

什么时候想到字符串套路：

- 题目要求处理连续子串、字符匹配、回文、前后缀或模式匹配
- 需要维护字符频次、左右边界或中心位置
- 字符顺序重要，不能随意排序
- 子串要求连续，子序列允许跳过字符

## 常用套路

- 中心扩展：处理回文子串，分别检查奇数中心和偶数中心
- 滑动窗口：处理最长/最短满足条件的连续子串
- 双指针：处理回文判断、反转、左右收缩
- 动态规划：处理回文子串、编辑距离、序列匹配等

## 易错点

- 区分子串和子序列：子串必须连续
- 回文串既可能是奇数长度，也可能是偶数长度
- 双指针访问字符前要先判断边界
- 扩展停止时，左右指针通常已经越过合法区间

## 面试表达

字符串题先确认目标是子串还是子序列，再判断是否需要维护边界、频次或匹配状态。回文子串类问题优先考虑中心扩展，思路清晰且 O(n^2) 通常足够。
